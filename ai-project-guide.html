웹·앱 UI/UX 디자이너 견적 산정 범위 가이드 보강판

화면 디자인 단가가 아니라
프로덕트 설계 범위와
전달 책임의 비용

핵심은 “예쁜 화면 몇 장”이 아니라 서비스 구조, UX 흐름, 상태값, 반응형, 디자인 시스템, 개발 핸드오프, 리스크 관리까지 어디까지 책임지는가입니다.

UI/UX 견적은 화면 수만으로 판단하지 않습니다. 복잡도, 기획 확정도, 구현 전달 수준, 검수 책임, 일정 리스크를 함께 봐야 합니다.

최종 문장

견적 산정의 기본 전제

웹·앱 UI/UX 디자인은 단순 시각 디자인이 아닙니다. 실제 업무는 서비스 목적 이해, 사용자 흐름 설계, IA와 정보구조 정리, 화면 구조 설계, UI 디자인, 컴포넌트와 상태값 설계, 반응형 설계, 접근성 고려, 개발 전달용 정리, 수정 대응, QA 또는 구현 피드백까지 포함할 수 있습니다.

UI/UX 견적은 구조·흐름·사용성·검증·전달까지 포함할 수 있는 전문 작업 기준으로 봐야 합니다.

NN/g가 UX를 정보구조, 인터랙션 디자인, 프로토타이핑, 사용자 테스트, UX 전략 등 디지털 제품 경험 개선 영역으로 다루는 것처럼, UI/UX는 화면 장식이 아니라 제품 설계 업무입니다.

화면 수보다 복잡도가 우선입니다

낮은 복잡도

회사소개, 단순 랜딩, 이벤트 페이지, 정적 브랜드 페이지, 정보성 페이지.

  • 기능이 적음
  • 상태값이 적음
  • 사용자 흐름이 단순함
  • 정책 판단이 적음

중간 복잡도

회원가입, 검색, 리스트/상세, 신청/예약, 마이페이지, 결제 전 단계, 콘텐츠 관리형 화면.

  • 입력값과 오류 상태가 있음
  • 필터, 정렬, 사용자 행동 흐름이 있음
  • 모바일 대응이 필요함

높은 복잡도

SaaS, 관리자, B2B 업무 시스템, 대시보드, 데이터 테이블, 권한 관리, 예약/매칭/정산, 캘린더 기반 서비스.

  • 정책 이해와 예외 케이스가 많음
  • 정보 구조와 상태값이 복잡함
  • 개발 협업 난이도가 높음
  • 디자인 시스템 필요성이 높음
화면 수가 같아도 UX 판단, 상태값, 정책, 데이터 구조, 반응형, 개발 전달 범위에 따라 견적은 크게 달라집니다.

반드시 분리해야 하는 견적 항목

기획 / 분석

기존 자료 분석, 요구사항 정리, 경쟁 서비스 조사, 기능 범위 파악, 사용자 니즈 정리, 화면 우선순위 설정, 문제 정의가 포함됩니다.

견적 영향 낮음

기획서, IA, 정책 문서가 확정된 상태

견적 영향 높음

아이디어만 있거나 디자이너가 기획을 재정리해야 하는 상태

자료가 많다는 것은 디자인 범위가 명확하다는 뜻이 아닙니다. 자료를 화면 구조로 재정리해야 한다면 UX 기획과 정보구조 설계가 별도 범위로 포함됩니다.

IA / UX 설계

  • IA 설계, 메뉴 구조, 사용자 플로우
  • 화면별 목적 정의, 정보 우선순위, 기능 흐름 설계
  • 예외 케이스와 정책 흐름 정의

NN/g가 정보구조를 내비게이션, 제품 내부 구조, 택소노미, 메타데이터 등으로 설명하듯 IA는 메뉴명이 아니라 사용자가 제품 안에서 정보를 찾고 이해하는 구조 전체입니다.

IA와 사용자 흐름이 없는 상태에서는 UI를 바로 그리는 것이 아니라 서비스 구조를 먼저 설계해야 합니다.

UI 디자인

주요 화면 디자인, 컴포넌트 적용, 레이아웃, 타이포그래피, 컬러 시스템, 콘텐츠 위계, 인터랙션 상태 반영이 포함됩니다.

  • 화면 수, 컴포넌트 수, 정보량
  • 상태값, 기능 복잡도, 시각 완성도
  • 브랜드 적용 난이도, 반응형 여부

상태값 설계

  • 기본, Hover, Active, Focus, Disabled
  • Loading, Empty, Error, Success
  • 권한 없음, 데이터 없음, 검색 결과 없음
  • 네트워크 오류, 입력 오류, 결제 실패
  • 승인 대기, 완료 상태
실제 서비스 화면은 기본 화면 하나로 끝나지 않습니다. 입력, 오류, 로딩, 빈 화면, 권한 상태까지 고려해야 하므로 상태값 설계는 별도 공수로 산정되어야 합니다.

반응형 설계

반응형은 PC 화면을 줄이는 작업이 아니라 디바이스별 사용성과 정보 우선순위를 다시 설계하는 작업입니다.

범위 구분

  • Desktop only
  • Desktop + Mobile 주요 화면
  • Full Responsive
  • Adaptive

포함 작업

  • 모바일 정보 우선순위와 내비게이션 재설계
  • 터치 영역, 이미지 크롭, 컴포넌트 재배치
  • 테이블/필터 모바일 대응, 브레이크포인트 정의

디자인 시스템 / UI Kit

디자인 시스템은 화면을 꾸미기 위한 참고 자료가 아니라 제품을 일관되게 확장하기 위한 운영 기준입니다.

  • 컬러, 타이포그래피, 8pt spacing, Grid
  • Button, Input, Select, Checkbox, Radio, Tab
  • Modal, Toast, Tooltip, Badge, Table, Pagination
  • Card, Empty state, Error state, Component variants
  • Usage rule, 토큰, 상태값, 확장성, 접근성 기준

접근성 / 사용성 기준

공공, 금융, 의료, 교육, SaaS, 글로벌 서비스에서는 접근성이 중요한 견적 요소가 됩니다. WCAG 2.2는 인지 가능, 운용 가능, 이해 가능, 견고함의 4가지 원칙과 A/AA/AAA 성공 기준을 다룹니다.

  • 명도 대비, 폰트 크기, 터치 영역, Focus 상태
  • 키보드 탐색, 오류 메시지, 폼 접근성
  • 이미지 대체 텍스트, 모바일 조작성 검토
접근성 기준까지 요구되는 경우 단순 시각 디자인을 넘어 검수 가능한 UI 기준이 필요하므로 별도 범위로 산정됩니다.

개발 핸드오프

  • Figma 파일 정리, 컴포넌트 네이밍, 상태값 정리
  • 간격 기준, 반응형 기준, 인터랙션 설명
  • 에셋 export, SVG 아이콘 정리, 개발자 질의 대응, 구현 QA
Figma 파일 제공과 개발 구현 가이드는 다릅니다. 개발자가 바로 구현할 수 있는 수준의 상태값, 반응형, 인터랙션, 컴포넌트 기준 정리는 별도 작업입니다.

국내 단가 자료를 볼 때 주의할 점

플랫폼 평균가의 한계

공개 플랫폼의 UI/UX 외주 가격은 단순 와이어프레임부터 커스텀 UI와 프로토타이핑까지 다양한 범위가 섞여 평균이 왜곡될 수 있습니다. 이 숫자를 웹앱, SaaS, 관리자, 디자인 시스템 작업의 기준 단가로 그대로 쓰면 안 됩니다.

실무 견적 기준

공공·산업 기준에서는 디자이너 등급별 노임단가가 별도로 공표됩니다. 실무 견적은 플랫폼 평균가보다 경력 등급, 투입 일수, 난이도, 책임 범위를 기준으로 보는 것이 안전합니다.

공개 플랫폼 평균가는 단순 외주 시장 참고용일 뿐, 시니어 UI/UX 디자이너의 웹앱·SaaS·디자인 시스템·개발 핸드오프 작업 단가를 대표하지 않습니다.

견적 산정 공식

기본 공식

총 견적 = 사전 분석 비용 + UX/IA 설계 비용 + UI 화면 디자인 비용 + 상태값 설계 비용 + 반응형 설계 비용 + 디자인 시스템 비용 + 그래픽/이미지/아이콘 제작 비용 + 프로토타입/인터랙션 비용 + 개발 핸드오프 비용 + 수정 대응 비용 + 일정 리스크 비용

실무 계산식

작업 공수 = 화면 수 × 화면 복잡도 × 상태값 개수 × 반응형 범위 × 기획 확정도 × 디자인 완성도 × 개발 전달 수준 × 수정 리스크

  • 기획 확정 / 자료 명확: 1.0
  • 자료 많지만 구조 미정: 1.3
  • 화면설계 없음: 1.5
  • 기능 정책 불명확: 1.5~2.0
  • 의사결정자 많음: 1.3~1.8
  • 빠른 마감: 1.5~2.0
  • 시안 2개 이상: 1.5~3.0
  • 반응형 전체 포함: 1.5~2.0
  • 디자인 시스템 포함: 1.5~3.0
  • 개발 QA 포함: 별도 산정

견적 패키지 예시

1단계

단순 UI 제작형

기획 확정, 와이어프레임 있음, 화면 수 적음, 반응형 제한적, 상태값 적은 프로젝트.

포함

  • 주요 화면 UI 디자인
  • 기본 스타일과 컴포넌트 사용
  • 1~2회 수정

비포함

  • UX 기획, IA 설계
  • 디자인 시스템, 전체 반응형, 개발 QA
2단계

UX/UI 설계형

웹앱, 모바일 앱, 신규 서비스, 화면 구조 개선이 필요한 프로젝트.

포함

  • 요구사항 분석, IA 정리
  • 주요 플로우 설계, UI 디자인
  • 주요 상태값, 기본 반응형 기준, Figma 정리

별도

  • 브랜드 아이덴티티
  • 고도화된 디자인 시스템, 전체 QA, 개발 문서화
3단계

프로덕트 디자인형

SaaS, B2B, 관리자, 대시보드, 데이터 기반 서비스.

포함

  • 서비스 구조 분석, IA/Flow
  • 전체 화면 UI, 상태값 설계
  • 컴포넌트 시스템, 반응형, 개발 핸드오프 기준

별도

  • UX 리서치, 사용자 테스트
  • 브랜드 시스템, 고급 인터랙션, 개발 퍼블리싱 가이드
4단계

브랜드 + 프로덕트 통합형

런칭 서비스, 투자용 데모, 브랜드 웹사이트 + 앱, 고완성도 마케팅 사이트, 글로벌 서비스.

포함

  • 브랜드 톤앤매너
  • UX/UI 설계와 디자인 시스템
  • 웹/앱 화면, 이미지/그래픽 제작
  • 인터랙션 가이드와 개발 전달 가이드

견적서에 반드시 넣어야 할 조항

1

범위 조항
본 견적은 합의된 화면 수, 기능 범위, 디자인 범위, 수정 범위 기준으로 산정됩니다. 범위 외 화면, 기능, 상태값, 반응형, 인터랙션, 기획 변경은 추가 비용이 발생할 수 있습니다.

2

기획 변경 조항
디자인 진행 중 기획, IA, 정책, 콘텐츠 구조가 변경될 경우 기존 작업 범위를 초과하는 재작업으로 간주합니다.

3

시안 조항
시안 수는 단순 화면 개수가 아니라 서로 다른 디자인 방향의 개수를 의미합니다. 시안이 추가될 경우 별도 비용과 일정이 발생합니다.

4

수정 조항
수정은 합의된 방향 내 보완을 의미합니다. 구조 변경, 기능 추가, 톤앤매너 변경, 신규 방향 제안은 추가 작업으로 산정됩니다.

5

반응형 조항
반응형 디자인은 별도 범위입니다. PC 화면을 단순 축소하는 것이 아니라 디바이스별 정보 구조와 사용성을 재설계하는 작업입니다.

6

개발 핸드오프 조항
Figma 파일 제공은 기본 산출물이며, 개발 구현을 위한 상세 가이드, 상태값 정의, 인터랙션 문서, QA 참여는 별도 범위입니다.

7

AI 사용 관련 조항
AI 도구 사용 여부는 작업자의 내부 제작 방식에 해당하며, 결과물의 책임·판단·수정·품질 검수·개발 전달 범위를 대체하지 않습니다. AI 활용으로 속도가 일부 개선될 수는 있으나 단가 인하 사유가 되지 않습니다.

  4. 콘텐츠와 카피는 확정되었는가?

  6. 디자인 시스템이 필요한가?
  7. 반응형 범위는 어디까지인가?
  8. 모바일 앱이면 iOS / Android 모두 필요한가?
  9. 로그인, 결제, 예약, 검색, 필터, 알림, 권한 기능이 있는가?
  10. 상태값 설계가 필요한가?
  11. 관리자 화면이 포함되는가?
  12. 프로토타입이나 인터랙션 가이드가 필요한가?
  13. 개발 핸드오프 문서와 개발 QA 참여가 필요한가?
  14. 시안은 몇 개, 수정은 몇 회 필요한가?
  15. 최종 산출물은 Figma만인지, 문서까지 포함인지?
  16. 일정은 언제까지이고 의사결정자는 몇 명인가?

견적 방어용 핵심 논리

Q

AI 쓰면 더 빨라지니까 싸져야 하는 것 아닌가요?

AI는 초안 생성, 이미지 생성, 레이아웃 아이디어, 코드 초안에는 도움을 줄 수 있습니다. 하지만 디자이너는 여전히 문제 정의, 구조 판단, UX 흐름, 브랜드 적합성, 화면 완성도, 수정 대응, 개발 전달, 최종 품질 검수를 책임집니다. AI가 도와주는 것과 단가가 낮아지는 것은 별개입니다.

Q

페이지 수가 적으니 간단하지 않나요?

페이지 수보다 중요한 것은 복잡도입니다. 한 화면 안에도 검색, 필터, 입력, 오류, 권한, 로딩, 반응형, 컴포넌트 상태가 포함되면 단순 페이지보다 훨씬 많은 설계가 필요합니다.

Q

일단 시안 여러 개 보고 결정하면 안 되나요?

시안은 단순 선택지가 아니라 각각 다른 전략과 구조를 가진 별도 디자인 방향입니다. 시안 수가 늘어나면 리서치, 레이아웃, 비주얼 시스템, 컴포넌트 판단이 모두 반복되므로 일정과 비용이 함께 늘어납니다.

Q

자료는 다 드렸으니 디자인만 하면 되지 않나요?

자료가 많다는 것과 기획이 정리되었다는 것은 다릅니다. 자료를 사용자 흐름, 정보 구조, 화면 위계로 재구성해야 한다면 UX 설계 작업이 포함됩니다.

최종 요약

웹·앱 UI/UX 디자인 견적은 화면 수, UX 복잡도, 기획 확정도, IA 필요 여부, 상태값 수, 반응형 범위, 디자인 시스템 범위, 접근성 기준, 개발 핸드오프 수준, 수정 리스크, 일정 리스크를 기준으로 산정해야 합니다.

웹·앱 UI/UX 디자인은 화면을 예쁘게 만드는 비용이 아니라, 서비스를 사용 가능한 구조로 설계하고 개발 가능한 제품 화면으로 정리하는 비용입니다.

특히 웹앱, SaaS, 관리자, B2B, 대시보드, 데이터 테이블, 복잡한 신청/예약/결제 플로우는 일반 웹페이지 단가가 아니라 프로덕트 디자인 비용으로 산정해야 합니다.
